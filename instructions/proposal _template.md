# 个人模块 Proposal 模板

> **提交格式**：请直接以 Markdown 文件提交，文件名建议为 `学号+姓名+proposal.md`。  
> **字数要求**：中文最多 8000 个字符；英文最多 25000 个字符。
> **写作重点**：请围绕个人负责模块展开，重点说明该模块解决的具体问题、技术实现路径、预期结果，以及该模块在整体系统中的定位、作用和与其他模块的连接融合方式。
> **说明**：*鼓励使用英文写作、使用图片表格进行讲述。此外，下面的提纲仅供参考，鼓励自由进行发挥。*

## 2. 项目整体背景与个人模块定位

### 2.1 项目整体目标

简要说明小组项目要解决的整体问题、目标用户或应用场景。

示例：

> 本项目面向 ______ 场景，尝试解决 ______ 问题。整体系统包括 ______、______、______ 等模块，最终希望实现 ______。

### 2.2 个人模块在整体系统中的定位（如果没有合并系统则不写）

请明确说明你负责的模块在整个系统中的位置和作用。

需要回答：

- 该模块属于系统中的哪一层或哪一部分？例如数据增强、模型架构优化等。
- 该模块接收哪些输入？
- 该模块输出哪些结果？
- 该模块为后续模块或整体系统提供了什么能力？

可配图说明模块位置，例如系统结构图、流程图、接口关系图。

## 3. 模块要解决的具体问题

请具体说明你的个人模块解决了什么问题，不要只写“完成某某功能”，而要说明该功能为什么必要。

建议从以下角度展开：

- 该模块面对的具体任务是什么？
- 原始问题中有哪些难点？例如数据质量差、识别准确率低、响应速度慢、模块之间数据格式不统一等。
- 如果没有该模块，整体系统会受到什么影响？
- 该模块的预期目标是什么？

示例：

> 本模块主要解决 ______ 问题。由于 ______，直接使用 ______ 方法会导致 ______。因此，本模块需要实现 ______。

## 7. 图片插入与提交要求

### 7.1 Markdown 图片插入方式

如果使用网络图片：

```markdown
![图片说明](https://example.com/image.png)
```

如果使用本地图片，建议在提交文件同级目录下建立 `images` 文件夹：

```text
项目提交文件夹/
  学号+姓名+proposal.md
  images/
    result.png
    system_architecture.png
```

然后在 Markdown 中使用相对路径插入：

```markdown
![系统结构图](images/system_architecture.png)
![运行结果截图](images/result.png)
```

### 7.2 本地图片提交要求

如果 Markdown 中引用了本地图片，提交时必须同时附带图片文件，否则老师或助教打开 Markdown 后将无法看到图片。

请确保：

- 图片路径使用相对路径，不要使用自己电脑上的绝对路径。
- 图片文件名尽量使用英文、数字或下划线，例如 `result_01.png`。
- 图片应清晰可读，能够说明系统结构、模块流程、运行结果或实验指标。
- 不要只提交截图而缺少文字解释，图片下方应有简短说明。
